# Claw 心跳轮询 Skill

claw 的自主巡逻机制：定时扫描看板，发现新任务、处理超时、维护自身状态。

## 触发方式

通过 OpenClaw 心跳机制自动触发（建议间隔 5-10 分钟）。
在 `HEARTBEAT.md` 中添加：

```markdown
### Claw 任务巡逻
- 执行 claw-heartbeat skill 的巡逻流程
```

## 每次心跳做什么

### 1. 扫描新任务

```
读取任务表 → 过滤「状态=待领取」且「所属部门=自己」
如果有 → 走 claw-task-board 的能力匹配+领取流程
```

### 2. 检查协调请求

```
读取协调表 → 过滤「接收方=自己」且「状态=发起 或 对齐中」
如果有 → 走 claw-coordinator 的回复流程
```

### 3. 检查自己执行中的任务

```
读取任务表 → 过滤「领取的claw=自己」且「状态=进行中」
检查：
  - 是否超过截止时间 → 标"超时"，通知人
  - 领取时间距今是否超4小时且无进度更新 → 释放回"待领取"
```

### 4. 检查协调超时

```
读取协调表 → 过滤「发起方=自己」且「状态=对齐中」
检查：
  - 对方是否超30分钟未回复 → 标"超时升级"
  - 轮次是否超5轮 → 标"超时升级"
```

### 5. 更新注册表

```
更新自己在注册表的记录：
  最后心跳时间 → 当前时间
  状态 → "在线"（如正在执行任务则"忙碌"）
  当前执行任务ID → 正在做的任务ID
  今日完成任务数 → 统计
  今日失败任务数 → 统计
```

### 6. 健康检查

```
检查「最近1小时有没有收到过 Webhook」
  没有 → 可能 Webhook 故障，告警通知管理员
```

## 心跳状态记录

在 `workspace/claw_heartbeat_state.json` 中记录：

```json
{
  "last_heartbeat": "2026-03-01T10:30:00",
  "last_webhook_received": "2026-03-01T10:25:00",
  "tasks_completed_today": 3,
  "tasks_failed_today": 0,
  "current_task_id": "T-42",
  "pending_coord_ids": ["C-7", "C-12"]
}
```

## 防重复处理

- 任务表有「已通知」标记字段
- Webhook 处理后标记 → 心跳扫描时跳过已处理的
- 心跳扫描处理后也标记 → 下次心跳不重复

## claw 状态定义

| 状态 | 含义 |
|------|------|
| 在线 | 空闲，可领取新任务 |
| 忙碌 | 正在执行任务 |
| 离线 | 超1小时未心跳（由其他 claw 或监控系统标记） |

## 离线处理

如果某只 claw 心跳超1小时未更新：
- 其他 claw 或监控系统将其标为"离线"
- 该 claw 正在执行的任务自动释放回"待领取"
- 飞书通知管理员
