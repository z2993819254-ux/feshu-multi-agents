# Claw 任务看板 Skill

让 claw 能够与飞书多维表格任务看板交互，实现完整的任务生命周期管理。

## 概念

基于「每个部门养一只 claw」的架构：
- **任务表**：项目经理创建任务 → claw 领取 → 执行 → 汇报
- **协调表**：跨部门对齐（见 `claw-coordinator` skill）
- **注册表**：每只 claw 声明自己的能力和状态

## 配置

每只 claw 在 `workspace/claw_config.json` 中配置：

```json
{
  "claw_id": "marketing-claw",
  "department": "营销",
  "bitable_app_token": "<飞书多维表格 app_token>",
  "task_table_id": "<任务表 table_id>",
  "coord_table_id": "<协调表 table_id>",
  "registry_table_id": "<注册表 table_id>",
  "skills": ["feishu-doc", "web-search", "xiaohongshu", "browser-automation"],
  "human_notify_target": "user:ou_xxxxxx"
}
```

## 任务状态流转

```
待分配 → 待领取 → 进行中 → 待审核 → 已完成
                     ↓          ↓
               已退回/需人工  已退回
```

## 操作步骤

### 1. 发现新任务

扫描任务表，过滤「状态=待领取」且「所属部门=自己部门」的记录。

```python
# 伪代码
records = feishu_bitable_list_records(app_token, task_table_id)
my_tasks = [r for r in records if r["状态"] == "待领取" and r["所属部门"] == MY_DEPT]
```

### 2. 能力匹配

读取任务「所需技能」，对比自身 skill 列表：

| 匹配结果 | 动作 |
|----------|------|
| **完全匹配** | 领取 → 状态改"进行中"，写入 claw_id 和领取时间 |
| **部分匹配**（缺非核心 skill） | 领取并标注"部分能力缺失: XXX" |
| **完全不匹配** | 不领取 → 状态改"需人工"，通知负责人 |

### 3. 执行任务

1. 读取「任务描述」，制定执行计划
2. 根据「任务类型」+「所需技能」调用对应工具
3. **每 30 分钟更新一次「执行摘要」**（让人看到进度）
4. 如果做不了 → 走退回流程

任务类型 → 执行方式：
- 文档撰写 → `feishu_doc` create/append
- 数据处理 → `exec` 脚本
- 调研分析 → `web_search` + 整理
- 代码开发 → `coding-agent` skill

### 4. 汇报结果

完成后更新字段：
- 状态 → "待审核"
- 执行摘要 → 100字内总结
- 产出物类型 → "文档链接" / "数据文件" / "代码PR" / "纯文本"
- 产出物链接 → URL
- 自评分数 → 1-10
- 遗留问题 → 未完成部分
- 完成时间 → 当前时间

### 5. 异常处理

| 异常 | 处理 |
|------|------|
| 执行失败（第1次） | 自动重试 |
| 执行失败（第2次） | 状态→"需人工"，通知人 |
| 需求不清/依赖缺失 | 状态→"已退回"，写退回原因，通知人 |
| 超过截止时间 | 标"超时"，通知人 |
| 领取超4小时无进度 | 释放回"待领取" |

### 6. 自评标准

| 分数 | 含义 |
|------|------|
| 9-10 | 完美完成，超出预期 |
| 7-8  | 完成，质量良好 |
| 5-6  | 基本完成，有小问题 |
| 3-4  | 部分完成，有遗留 |
| 1-2  | 几乎没完成 |

## 人工介入触发条件（铁律）

以下情况 **必须** 通知人工，claw 不能自己做决定：
- 任务执行失败 2 次
- 自评信心低于 6 分
- 涉及金额 / 合同 / 对外发布
- 需求变更或优先级调整

以下情况 claw **可以** 自主处理：
- 纯技术细节对齐（格式、参数、接口）
- 标准化任务（有 skill 支持的）
- 文档整理、数据查询等常规操作
