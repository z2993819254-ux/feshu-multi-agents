# Claw 跨部门协调 Skill

处理跨部门对齐：异步（飞书协调表）+ 实时（sessions_send）。

## 什么时候需要协调

- 任务依赖其他部门的产出物（如：营销需要设计部的素材）
- 任务涉及多部门配合（如：产品需求需要研发+设计同时介入）
- 技术方案需要跨部门确认（如：接口格式对齐）

## 方式一：异步协调（协调表）

适用场景：不紧急、可以等几小时。

### 发起对齐

在协调表新建记录：
```
关联任务ID → 当前任务ID
发起方 → 自己的 claw_id
接收方 → 目标部门的 claw_id
问题描述 → 具体问题
对齐状态 → "发起"
当前轮次 → 0
创建时间 → 当前时间
```

### 回复对齐

发现协调表中「接收方=自己」且「状态=发起 或 对齐中」的记录：

1. 读取问题描述和对话记录
2. 追加回复到「对话记录」字段：`[{claw_id} {HH:MM}] 回复内容`
3. 更新「当前轮次」+1
4. 如果已解决 → 状态改 "已解决"，填写「结论」
5. 如果需要继续 → 状态改 "对齐中"

### 对话格式

```
[marketing-claw 14:30] 我们需要首页 Banner 的设计稿，尺寸 1920x600，3月5日前
[design-claw 14:45] 收到，确认尺寸。需要提供文案和产品图吗？
[marketing-claw 15:00] 文案已在任务描述里，产品图见附件链接 xxx
[design-claw 15:10] OK，预计3月3日交付初稿
```

## 方式二：实时协调（sessions_send）

适用场景：紧急、需要快速对齐。

```python
# 发起实时对话
sessions_send(
    label="design-claw",  # 或用 sessionKey
    message="【紧急对齐】任务#42 的 Banner 设计，客户催得急，能否今天出初稿？"
)

# 等待回复（对方 claw 会通过 sessions_send 回复）
```

**重要：** 实时对话结论必须写回协调表，保证人能看到记录。

## 自动升级规则（铁律）

| 条件 | 动作 |
|------|------|
| 对话超过 **5 轮** 未解决 | 状态→"超时升级"，飞书通知双方部门负责人 |
| 一方超过 **30 分钟** 无回复 | 状态→"超时升级"，通知人工 |
| 涉及 **需求变更/优先级调整** | 直接通知人工，claw 不做决定 |
| 涉及 **金额/合同/对外发布** | 直接通知人工 |

## claw 可自主对齐的范围

- 技术细节：接口格式、参数命名、数据结构
- 排期微调：在截止日期前的内部调整
- 文件交接：产出物传递、格式确认

## 通知人工

```python
# 通过飞书消息通知
message(
    action="send",
    target="user:ou_xxxxx",
    message="⚠️ 跨部门对齐超时升级\n任务#42 - Banner设计\n营销claw ↔ 设计claw 已对话5轮未解决\n请人工介入"
)
```
